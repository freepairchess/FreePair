# FreePair v0.27

A pairing-sheet and tournament-setup release. Each section now has a clear
**pairing style** that drives the buttons you see and locks once you start, and
the **Advanced Options** dialog gains a big set of column-customisation tools —
custom headers, one-cell player blocks, blank write-in score boxes, and a
print-ready wall-posting sheet by default.

## What's new in v0.27

- **Each section has a pairing style that drives the buttons.** A section's
  **Pairing rule** (Swiss, Round Robin, Quad, …) is now the single setting that
  says how it's meant to be paired. The pair button follows it — **"Pair next
  round"** for Swiss, **"Pair as Round-Robin"**, or **"Pair as Quads"** — so you
  always see the right action. "Pair as Quads" and "Pair as Round-Robin" set the
  style on the new sections for you.

- **The pairing style locks once round 1 is paired.** Before the first round you
  can change it freely; afterwards it's read-only, so an event can't drift
  mid-tournament. If you ask for a Swiss pairing on a round-robin or quad
  section (or pair a Swiss section as quads / a round robin), FreePair asks you
  to confirm first instead of doing something unexpected.

- **Rename any pairing column.** In **Advanced Options**, every column now has a
  box where you can type a custom header — for example show **White Rating** as
  **`W-Rtg`**. Leave it blank to keep the default. The on-screen grid and the
  printed PDF each keep their own names.

- **One-cell "player block" columns.** New optional **White Player** /
  **Black Player** columns combine a player's title, name, rating, score, colour
  history and colour-due into a single cell —
  `FM Castaneda, Nelson 1850 [4.0 WBW w]` — for a classic, compact pairing-sheet
  look. Tick them on (and untick the separate columns) whenever you prefer it.

- **Print-ready wall-posting score sheets.** Two new **write-in result** columns
  print as blank boxes for players to fill in by hand, and the **printed PDF now
  comes set up as a score sheet by default**: board number, a write-in box and a
  player block for each side. Don't like the default? **Restore Default** brings
  it back any time, and **Copy from grid** makes the printout match your screen.

- **Scores print with one decimal.** Printed pairing sheets now show **`1.0`**,
  **`0.5`**, **`2.0`** (instead of `1`), so a hand-written half-point can't be
  mistaken for the printed score.

- **See your layout before you print.** Advanced Options now shows a **live
  sample row** for both the on-screen grid and the printed PDF, updating as you
  show / hide, reorder, or rename columns — so you can dial in exactly what you
  want. The dialog is also more compact and fits smaller screens.

- **Pairing controls on the Pairings tab.** The pairing engine (USCF / FIDE) and
  the section's pairing rule now sit right on the Pairings toolbar (read-only
  once you've started pairing), the left section list can be resized, and long
  event / section names show in full on hover.

## Installer

- **Download:** [FreePair-win-x64-Setup.exe](https://github.com/freepairchess/FreePair/releases/latest/download/FreePair-win-x64-Setup.exe)
- **Portable zip:** [FreePair-win-x64-Portable.zip](https://github.com/freepairchess/FreePair/releases/latest/download/FreePair-win-x64-Portable.zip)
- **Auto-update:** Existing FreePair installs (v0.3 and later) pick up v0.27
  automatically. To update now, click "Check for updates" in the Settings tab.

The installer bundles the .NET 10 runtime, the BBP Pairings FIDE engine, and
the rules-reference document — no additional downloads or configuration
required.

## Known limitations (deliberate trade-offs, planned for later)

- **The pairing style currently controls intent and the buttons, not a second
  pairing engine.** Choosing Round Robin or Quad gives you the right workflow,
  the right buttons, and the locking behaviour; the underlying schedule a
  section produces is unchanged from before. Tightening every legacy import so
  the engine always follows the displayed style is a planned follow-up.

- **Column choices are app-wide preferences.** The columns you pick (and any
  custom names) apply across your events on this computer, not per-event — so a
  layout you set up for one tournament is ready for the next.

- **A handful of late-round colour and pairing choices remain at the TD's
  discretion.** Validated against a large real-world SwissSys corpus, FreePair
  reproduces the great majority of pairings exactly and matches 100% on many
  representative events. A small number of genuinely discretionary situations
  — where two pairings are equally valid under the rule book and SwissSys picks
  one by convention — are documented and left to the TD rather than forced to a
  single interpretation.
