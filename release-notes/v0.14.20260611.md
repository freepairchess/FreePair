# FreePair v0.14.20260611

FreePair is a free, open-source desktop application for chess-tournament directors. It imports SwissSys 11 `.sjson` files, runs USCF Swiss and FIDE Dutch pairings (via FreePair's own USCF engine and the BBP Pairings FIDE engine, both bundled), and gives the TD end-to-end tools to pair, score, intervene, and publish reports.

This release sharpens the **printed reports for posting on the wall** and gives the **Event configuration** screen a cleaner, tighter layout.

## What's new in v0.14

- **Cleaner, more compact Event configuration.** The settings screen has been reorganised into **Event Settings** (time control, format, type, pairing rule, rounds, half-point byes, rating system, pairing engine) and **Organizer and NAChessHub Settings**, with tighter spacing, evenly-sized fields, and a compact date picker so start and end dates sit on one line. The **NACH Event ID** is now a clickable link straight to your event's public NA Chess Hub page.
- **Pairing sheets built for the wall.** The Pairings PDF now reads the way players expect on a posted sheet: each player shows as **name with rating** (e.g. "GM Cheng, Tyler (2571)"), there are clean **grid lines** around every cell, and the pair number is dropped so the sheet is just board, players, ratings, and a space for the result.
- **Tighter report headers.** Every report now leads with the **event name and section on one line** (e.g. "Chess A2Z June Open 2026 - Section : Open I"), with the time control and dates on a single line beneath — less wasted space at the top of each printout.
- **Standings and wall charts gain grid lines** for easier reading at a glance, and the pair-number column is now clearly labelled **"Pair #"** so it's understood as each player's fixed entry number, not a live ranking. The **Place** column reflects the same score-based standing you see on screen.
- **Clearer seeding controls.** On the Players tab, the seed up/down arrows now live under a **"Change Pair Number"** heading, and the first and last players show their (unavailable) arrow greyed-out so the column lines up consistently.
- **Smarter seeding for players added before round 1.** When you add a player before the first round is paired, players with the **same rating are now ordered alphabetically by name** — the same rule the "Restore Default Pairing Number" button uses — so a late addition lands in the right place instead of at the bottom of its rating group. (Players added mid-event still keep a fixed entry number, since renumbering would scramble already-played rounds.)

## Known limitations (still planned for later releases)

- **Event-level classification is blank when the source file doesn't carry it.** SwissSys records some classification details per section rather than at the event level; when the imported file has no event-level values, the Event Settings dropdowns start blank and the TD can set them (they then save with the event).
- **Some discretionary colour and float choices still differ from SwissSys.** A handful of late-round situations turn on which player floats between score groups, or on a colour call SwissSys makes without a clear rule basis; FreePair follows the stricter USCF reading in those cases. These are validated against a large real-world SwissSys corpus and tracked as deliberate, documented trade-offs.
- **Cross-half colour interchange (USCF 29E5d) remains present but switched off**, pending a validated rule for when SwissSys takes an interchange versus tolerating a colour clash.
- **Quads are formed by rating order.** The one-click split seeds strictly by rating; there's no manual re-seeding of who lands in which quad yet.
