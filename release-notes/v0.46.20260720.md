# FreePair v0.46

A **feature** release that adds **Bughouse team-event support** end to end.
Nothing changes for your existing events — every non-Bughouse section pairs,
scores, and reports exactly as before — so upgrading is low-risk and
recommended.

## What's new in v0.46

### Bughouse events

- **Choose the event's chess variant.** Event configuration (and the
  Create-new-event dialog) now has a **Chess Variant** setting — **Classical**
  (the default), **Chess960**, or **Bughouse**. If your event file already
  specifies a variant, FreePair honors it; otherwise it stays Classical.
- **Turn a regular roster into a Bughouse event in one click.** When the event
  variant is Bughouse, each section gets a **Make bughouse section** button.
  Assign every player a two-person team (team names are matched
  case-insensitively and tidied to upper-case), click the button, and FreePair
  builds a new **Bughouse section** whose entrants are the teams: the team name
  becomes the competitor, the two players' ratings are **added together** for
  pairing, and both players' names are shown together. The new section is a
  **Double Swiss** ready to pair. If a team is missing a player — or a player is
  missing a team — FreePair explains exactly what to fix.
- **Open a Bughouse section straight from your registration provider.** If your
  event file already contains a calculated Bughouse section, FreePair recognizes
  it and lets you **pair it directly** — no extra setup.
- **The app keeps Bughouse sections safe.** For a calculated Bughouse section,
  FreePair shows a banner and hides the actions that don't apply — remaking the
  section, syncing its roster online, verifying IDs, refreshing ratings, or
  searching the player database — because its "players" are really teams. You can
  toggle a section's Bughouse status right on its Overview tab.

### Smoother event & section setup

- **Create-event now flows into section setup.** The Create-new-event dialog
  carries all the event-level settings (dates, location, format, rating system,
  pairing rule, chess variant, engine, rounds, and half-point-bye allowance).
  After you create the event, FreePair opens a **section dialog** pre-filled with
  those settings so your first section inherits them. Opening an event that has
  no sections yet does the same.
- **Less clutter when adding a section.** The redundant "Kind" choice is gone —
  the pairing rule now determines everything the section needs.

### Quality-of-life

- **Recent-tournaments list is easier to read.** The dialog is wider, file names
  are shown in full, and the "Remove" button is no longer clipped.

- **Download:** [FreePair-win-x64-Setup.exe](https://github.com/freepairchess/FreePair/releases/latest/download/FreePair-win-x64-Setup.exe)

- **Auto-update:** Existing FreePair installs will pick up v0.46 automatically.
  To update now, click "Check for updates" in the Settings tab.

The installer bundles the .NET 10 runtime, the pairing engines, and the
rules-reference document — no additional downloads or configuration required.

## Known limitations

- **SwissSys fidelity is unchanged.** The Swiss pairing engine and its
  regression coverage are exactly as in the previous version; the known
  divergences remain catalogued as deliberate future-work trade-offs.
- **Bughouse pairing uses the standard Double-Swiss logic** over the combined
  team entries; individual-board results within a team are still recorded on your
  own score sheets.
