# FreePair v0.22

A tournament-management quality-of-life release focused on running
multi-section and dual-rated (USCF + FIDE) events. You can now copy a
section, move players between sections, verify player IDs against the US
Chess ratings service before round 1, and re-open recent tournaments in a
couple of clicks. The pairing engines (USCF and FIDE) are unchanged.

## What's new in v0.22

- **Copy a section.** Duplicate any section into a new one — handy for
  splitting an over-full section or seeding a reserve. A dialog lets you
  choose whether to carry over each player's requested byes and whether to
  copy the played rounds (pairings + results) or start the copy fresh and
  unpaired. Player identities, ratings, and seeding always come across.

- **Move players to another section.** Select one or more players and move
  them into another section — existing or brand-new. If the destination has
  already paired rounds, FreePair asks what to record for the rounds the
  moved players missed (full-, half-, or zero-point bye) **per round**, so
  the destination's standings stay consistent.

- **Verify player IDs before round 1.** A new "Verify IDs" button checks each
  player's ID against the US Chess ratings service and marks it with a green
  check (verified) or a red "!" (needs attention) — with a hover tooltip
  explaining what's wrong. A small progress dialog shows "X / N verified"
  while it runs. Verification is **advisory only**: it never blocks pairing,
  and it degrades gracefully when you're offline.

- **Smart handling of dual-rated (USCF + FIDE) files.** For events that carry
  both a national ID and a FIDE ID, FreePair figures out which column is
  which and labels them automatically — "ID [USCF]", "ID2 [FIDE]" — and shows
  the same for the rating columns ("Rating [Pairing Rating]" plus an optional
  "Rating2"). Verified IDs become clickable links to the player's US Chess or
  FIDE profile. If a file has its ID (and rating) columns reversed, a
  one-click "Switch Player IDs" fixes the whole section at once — and asks
  first whether to swap the ratings too, with a guard that keeps a player's
  usable rating for pairing when the other rating is 0 (e.g. a US Chess player
  with no FIDE rating).

- **Open recent tournaments.** A new "Open recent…" button lists the
  tournaments you've opened before, with an Open button, the event name, and
  the file name (click it to jump to the file in its folder; hover to see the
  full path). No more digging through folders to reopen last week's event.

- **Easier-to-read tooltips.** Tooltips across the app now use a high-contrast
  colour scheme that stays legible in both light and dark themes.

## Installer

- **Download:** [FreePair-win-x64-Setup.exe](https://github.com/freepairchess/FreePair/releases/latest/download/FreePair-win-x64-Setup.exe)
- **Portable zip:** [FreePair-win-x64-Portable.zip](https://github.com/freepairchess/FreePair/releases/latest/download/FreePair-win-x64-Portable.zip)
- **Auto-update:** Existing FreePair installs (v0.3 and later) pick up v0.22
  automatically. To update now, click "Check for updates" in the Settings tab.

The installer bundles the .NET 10 runtime, the BBP Pairings FIDE engine, and
the rules-reference document — no additional downloads or configuration
required.

## Known limitations (deliberate trade-offs, planned for later)

- **ID verification is US Chess only for now.** The green-check / red-"!"
  verification checks IDs against the US Chess ratings service and reads back
  the linked FIDE ID from there. A direct FIDE lookup (and other national
  federations) is planned; until then, FIDE IDs are confirmed via the linked
  US Chess record rather than queried independently.

- **Moving players is a pre-pairing tool on the source side.** Players can be
  moved out of a section before its round 1 is paired. Once a section has
  paired a round, withdraw the player there instead — moving them would break
  the crosstable references in the rounds they already played.
