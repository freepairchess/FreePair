# FreePair v0.33

A follow-on to v0.32's mid-event work. This release makes **half-point byes**
smarter for players who join after the event has started, and lets each
**section set its own half-point-bye limit** instead of sharing a single
event-wide number.

## What's new in v0.33

- **Late entries who didn't ask for byes now start with the right score.**
  Previously, when a mid-event sync added a player who had *not* requested any
  byes, FreePair recorded a **zero-point bye for every round they missed**. Now
  those missed rounds are pre-filled with **half-point byes up to the section's
  allowance**, and only the rounds beyond the allowance fall back to zero-point.
  A player who arrives for round 4 of a 2-bye event, having requested nothing,
  now starts on **1.0** (two half-point byes) instead of 0 — the same result a
  TD would enter by hand. As always, every round is a drop-down you can change
  before you accept.

- **Each section can set its own half-point-bye limit.** A new **"Section
  half-point byes"** panel on the Event configuration tab lets you give any
  section its own allowance. Tick **Override** and choose a number to pin that
  section; leave it unticked and the section simply **inherits the event-level
  allowance**, exactly as before. This is ideal for mixed events — for example a
  scholastic section that permits more byes than the championship section
  alongside it.

- **The section limit drives the late-entry defaults too.** When you assign
  missed-round results for a mid-event arrival, the half-point byes FreePair
  pre-fills now respect **that section's** allowance rather than only the
  event-wide number — so a section that allows more (or fewer) byes gets the
  matching default for late entries.

- **Your section limits travel with the file.** A per-section override is saved
  in the tournament file and comes back when you reopen it. Sections that just
  inherit the event allowance add nothing to the file, so nothing changes for
  events that don't need per-section limits.

required.

## Known limitations (deliberate trade-offs, planned for later)

- **No allowance recorded still means zero-point byes.** If neither the section
  nor the event records a half-point-bye allowance, a late entry's missed rounds
  default to zero-point byes — the safe default for a round they didn't play.
  Set a section or event allowance to have half-point byes pre-filled instead;
  you can always change any round by hand.

- **The section limit governs the pre-filled defaults, not a hard cap on your
  edits.** The allowance decides what FreePair *suggests* for missed rounds. You
  remain free to override any individual round up or down when you assign
  results — the number is there to save you typing, not to lock your hand.

- **Adding a mid-event player still leaves the pairing to you.** As before,
  FreePair records the newcomer and their missed-round results; you then pair the
  next round normally. The new player carries no colour history from the byes, so
  their first game's colour is assigned like any other unpaired entrant.
