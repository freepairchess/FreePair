# FreePair v0.15.20260611

FreePair is a free, open-source desktop application for chess-tournament directors. It imports SwissSys 11 `.sjson` files, runs USCF Swiss and FIDE Dutch pairings (via FreePair's own USCF engine and the BBP Pairings FIDE engine, both bundled), and gives the TD end-to-end tools to pair, score, intervene, and publish reports.

This release is all about the **pairing screen**: clearer byes, a "why was this paired?" explanation on every line, and a smoother, more readable layout.

## What's new in v0.15

- **Byes now appear right on the pairing list.** Instead of a separate panel, byes show at the **bottom of the pairings with board number 0** — the way they appear on a posted sheet — with the player's points and a short note ("Requested half-point bye", "Requested zero-point bye", or "See TD if you have questions" for a full-point bye). This shows on screen, in the preview before you commit, and on the printed Pairings PDF.
- **Change a bye on the spot.** You can switch a bye between full / half / zero point directly on the pairing line. FreePair asks you to confirm first, since it changes the player's score and standings.
- **"Why this pairing?" — now for byes too.** The per-line **?** button explains how a board was paired, and now also explains **why a player got a bye** (a requested half/zero-point bye, or the engine assigning the round's full-point bye).
- **See each player's form at a glance in the preview.** When you pair a round, the preview now shows each player's **points, rating, and colour history** next to their name (e.g. "Zhang, Timothy [1.0, 386, W]"), so you can sanity-check the matchups before committing.
- **A heads-up while pairings are being calculated.** Clicking "Pair next round" now shows a clear **"Calculating pairings…"** indicator so you know the engine is working before the preview appears.
- **Hand-edited pairings are clearly marked.** If you change a pairing in the preview (swap colours, swap opponents, move a bye), FreePair lets you add an optional note and flags the board as a **"TD Manual Override"** in the explanation — while still showing what the engine had originally chosen and why.
- **Easier-to-read tables.** Across Pairings, Standings, and Wall Chart, column headers now show their **full names** (Board, Rating, Solkoff, Cumulative, …) and the player-name columns size to the name instead of leaving a large empty gap. The result drop-down was widened so bye labels are no longer cut off.
- **Your column widths stick.** Resizing a column and then entering a result (or changing a bye) no longer snaps the table back to its default widths.
- **Cleaner navigation.** Opening a tournament now lands on the **Event configuration** screen so you can review the event details first, and the highlight for the selected event / section now uses your **Windows accent colour** consistently, with readable text.

## Known limitations (still planned for later releases)

- **Editing a bye is a single-line change, not a re-pair.** Switching a bye's point value updates the player's score; it doesn't pull them back into a game against an opponent (there's no opponent to restore mid-round).
- **Some discretionary colour and float choices still differ from SwissSys.** A handful of late-round situations turn on which player floats between score groups, or on a colour call SwissSys makes without a clear rule basis; FreePair follows the stricter USCF reading in those cases. These are validated against a large real-world SwissSys corpus and tracked as deliberate, documented trade-offs.
- **Cross-half colour interchange (USCF 29E5d) remains present but switched off**, pending a validated rule for when SwissSys takes an interchange versus tolerating a colour clash.
- **Quads are formed by rating order.** The one-click split seeds strictly by rating; there's no manual re-seeding of who lands in which quad yet.
