# FreePair v0.40

This release focuses on **adding players faster**, **fixing byes at any point
in the event**, and a **cleaner, roomier left-hand navigation** — plus Event
Details that now save themselves.

## What's new in v0.40

- **Add players from an online database.** The Roster tab's **➕ Add** button now
  offers two paths: *Add player manually* or *Search from player DB*. The search
  looks players up by name or ID number, shows the matches with their Title,
  federation, IDs and ratings (USCF / FIDE / CFC), and lets you click **Add** on
  the right person to drop their details straight into the Add-player form — where
  you can still set byes for past rounds before saving. Each ID in the results is a
  **clickable link** to that player's public USCF / FIDE / CFC profile so you can
  confirm you've got the right person.

- **Richer player details.** The Add / Edit player dialog gained an **Optional Info**
  tab for Title, FIDE ID, FIDE country, sex, birth date, per-federation IDs and
  ratings, and check-in — so a manually entered (or searched) player can carry the
  same detail as an imported one. When you pick a player from the search, FreePair
  even routes their IDs and ratings into the right ID / ID2 and Rating / Rating2
  slots based on the section's rating system.

- **Fix a missed-round bye at any time.** When a player joins (or is moved into) a
  section after rounds are already paired, you can now open **Edit player** and set
  or correct the bye for those **already-paired rounds** right from the Roster —
  no need to catch it during the move. Change a zero-point to a half-point (or full)
  and the score, standings, and wall chart update to match.

- **NA Chess Hub bye changes stick.** Fixed a case where syncing a bye that changed
  from zero-point to half-point (e.g. after the organizer set the allowed byes on
  NA Chess Hub) showed the change but didn't actually reach the Byes & Withdrawals
  list. The corrected bye now lands where it should.

- **Event Details save themselves.** The Event configuration form no longer has an
  **Apply** button — every change now saves automatically as you make it, just like
  the rest of FreePair. No more lost edits from forgetting to click Apply.

- **A roomier sections list.** For events with many sections the left pane is now far
  easier to work with:
  - **Collapse / expand each section** (collapsed shows just the name); **Expand all**
	/ **Collapse all** buttons at the top; selecting a section auto-expands it.
  - **Empty sections are flagged** with an "empty" badge and start collapsed, so they
	stay out of the way while still catching your eye.
  - **Drag to reorder** sections with the grip handle — no more repeated clicks on
	the up / down arrows.
  - **Event Operations** (Pair All, Sync All, Publish, Exports, …) tuck into a
	collapsible section so the roster list gets the space.

- **Small polish.** The Event Details date fields are wider and clearer with a proper
  calendar button, the unused Time zone field was removed, and a "Fall back to US
  Chess search" option plus the player-database endpoint are configurable in Settings.

  in the Settings tab.

The installer bundles the .NET 10 runtime, the BBP Pairings FIDE engine,
and the rules-reference document — no additional downloads or configuration
required.

## Known limitations (still planned for later releases)

- **Online search needs an internet connection.** When the player database is
  offline, FreePair can fall back to a US Chess search (a Settings toggle); with no
  connection at all, add players manually as before.
- **Optional player fields are display-oriented.** The extra IDs and ratings enrich
  the roster and reports; pairing still uses the section's pairing rating.
- **Section collapse state is per session.** How you've expanded or collapsed the
  sections list isn't saved with the event file yet.
