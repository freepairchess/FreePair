# FreePair v0.4.20260604

In-app version display and on-demand update check, on top of the v0.3.20260604 USCF pairing engine improvements.

## What's new

### See your version, check for updates on demand

A new row in **Settings → Auto-update** shows the running FreePair version (read live from the assembly metadata) plus a **Check now** button that polls GitHub Releases on demand and reports the result inline.

Three outcomes you might see:

- **FreePair is up to date (v…).** Nothing to do.
- **Update available: v… — See the banner at the top of the window.** The main-window banner appears with View notes / Update now buttons.
- **Update check failed: …** or **Updater not available in this build (dev / portable run).** Tells you *why* nothing happened instead of leaving you guessing.

This closes a UX gap: previously, the auto-update banner could silently never appear because the startup check returned NotInstalled (dev builds) or Failed (network / GitHub issue), and there was no way for a TD to tell the difference from "I'm already on the latest". Now you can confirm.

### Beta channel (unchanged)

The **Include pre-release builds (beta channel)** checkbox is left off by default. FreePair has never published a pre-release, so it currently has no effect; it's a forward-looking option so future preview builds can opt-in TDs who want to test new features ahead of stable. Live-tournament TDs should leave it off.

## Pairing engine

No changes since v0.3.20260604. All of that release's improvements ship here too:

- Better float-down picks under colour pressure (drop-selection considers the true minimum colour cost per candidate).
- Equalisation wins over score when colour preferences match.
- Smarter alternation tiebreaker by per-colour recency.
- Unified rule for both-no-history pairings (higher-rated gets White, regardless of score).

See v0.3.20260604's notes for the full breakdown.

## Installer

Self-contained Windows x64 installer (`FreePair-win-x64-Setup.exe`) and a no-install portable build (`FreePair-win-x64-Portable.zip`). Both bundle:

- the FreePair desktop application (Avalonia UI),
- the FreePair USCF pairing engine,
- the FIDE Dutch pairing engine (BBP Pairings),
- the .NET 10 runtime — **no separate runtime install required**.

The auto-updater (Velopack) is included. From v0.4.20260604 onward, you can confirm the in-app version and trigger an on-demand check from `Settings → Auto-update`.

## Known limitations

Unchanged from v0.3.20260604:

1. **Board-ordering preferences** — pairings, colours, and bye all match SwissSys; only the board numbering differs.
2. **SwissSys-discretionary choices that conflict with USCF rules** — SwissSys sometimes gives byes to unrated players when a rated alternative was available, or floats a non-lowest-rated player; FreePair correctly follows USCF rules. See `docs/USCF_DISCRETIONARY_DIVERGENCES.md`.
3. **Deferred engine improvements requiring larger refactors** — weighted conflict-cost model, absolute-board-number threading, and floater-lock heuristic refinements. Catalogued in the same doc.
