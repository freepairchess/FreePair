# FreePair v0.38

This release focuses on **the Roster tab, player check-in, and NA Chess Hub
sync** — the tools you use to get an event ready and keep it in step with
online registration — plus a wave of polish that makes every grid and report
easier to read.

## What's new in v0.38

- **Player check-in.** Turn on **Require check-in** for the whole event or a
  single section, and each player gets a check-in toggle on the Roster tab.
  Players who haven't checked in are flagged with a red marker, and FreePair
  warns you before pairing if anyone still isn't checked in — so late arrivals
  and no-shows don't slip into the pairings by accident. The check-in column
  is sortable, and toggling a player's status keeps the roster order stable.

- **Check-in status now syncs from NA Chess Hub.** When you **Sync Roster
  with NACH**, the review dialog now also shows players whose check-in status
  changed online, alongside the new registrations and bye changes it already
  found. Tick the ones you want and FreePair updates them — so a player who
  checks in on the website shows up as checked in for pairing. (Only shown for
  sections that require check-in.)

- **Fixed: Roster action buttons that did nothing.** **Sync Roster with
  NACH**, **Verify IDs**, **Refresh Ratings**, and **Switch Player IDs** could
  silently do nothing when clicked. They now work as intended.

- **Edit a section's details right on the Overview tab.** A section's **name**
  and **time control** are now editable inline on the Overview tab — no need to
  open the Edit dialog for a quick fix. The Overview controls are laid out in a
  natural order (name, time control, rounds, half-point-bye allowance, require
  check-in), and the same fields are editable inline: rounds, the per-section
  half-point-bye allowance, and the check-in requirement.

- **Renamed "Players" to "Roster."** The tab you use to manage entries is now
  called **Roster**, matching how TDs talk about it.

- **Roomier, easier-to-read grids everywhere.** Columns across the Roster,
  Standings, Wall Chart, and Byes tabs now auto-size to their contents so
  values (names, emails, phones, teams) are no longer clipped, and inline
  **copy icons** let you grab a value with one click. Grids show full cell
  borders with bold, themed headers and a subtle per-theme tint, so rows are
  easier to follow. Action icons (delete / edit) moved to the first column,
  the pair-number and "change pair number" controls were merged into one
  compact column (hidden once a round is paired), and byes get their own
  resizable grid with a **Print as PDF** button.

- **Per-round result columns in Standings and Wall Chart.** The stacked
  "Rounds" column is now split into individual **R1…RN** columns on both the
  main and pop-out views, so you can read each player's round-by-round results
  straight across. Pop-out Standings also gained the Byes column, and
  calculated prizes are now included in the Standings PDF.

- **FreePair branding on every PDF.** All reports now carry the FreePair logo
  and the app version, so a printout is easy to identify and trace back to the
  build that produced it.

## Known limitations (still planned for later releases)

- **Check-in syncs from NACH only when a section requires it.** If a section
  doesn't have **Require check-in** turned on, check-in differences aren't
  surfaced in the sync review — by design, so events that don't gate on
  check-in stay uncluttered.
- **NACH sync covers registrations, byes, and check-in.** It deliberately
  never touches pairings, results, ratings, or contact edits — you stay in
  control of those.
- **Tiebreak selection is per section.** Each section keeps its own tiebreak
  configuration; there is no single event-wide setting yet.
