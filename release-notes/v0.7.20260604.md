# FreePair v0.7.20260604

FreePair is a cross-platform chess-tournament pairing application for Windows. It imports SwissSys 11 `.sjson` files, runs USCF Swiss and FIDE Dutch pairings (via FreePair's own USCF engine and the BBP Pairings FIDE engine, both bundled), and gives the TD end-to-end tools to pair, score, intervene, and publish reports.

This release is a transparency pass on the USCF pairing engine: every pairing decision is now explained, cited, and visualised on demand.

## What's new in v0.7

- **"Why this pairing?" dialog on every Pairings row.** Each board now has a small **?** button next to it. Click it to open a modal that explains exactly how the engine arrived at that pair: a header with the section, round, and both players (name + rating); a **Pairing reason** block listing each matching step (natural slide, transposition, interchange, colour-driven swap, forced merge); a **Colour assignment** block explaining who got White and why; and a **Round context** block listing the round-level decisions that shaped the score group (floater drops, byes, forced merges). When nothing unusual happened, the dialog shows a calm "✓ Natural pairing" badge instead.
- **SwissSys-style switch diagram.** Below the text reasons, the dialog renders the entire score group containing this board the way SwissSys does: top half on the left, bottom half on the right (rating-desc), each player labelled with their SwissSys colour-due code (`w`/`W`/`WW`/`b`/`B`/`BB`), connected by a straight line per natural slide pair and a purple diagonal per transposition or interchange. At a glance you can see which players were swapped to produce the pairing you're looking at.
- **USCF rule citations on every annotation.** Every pairing decision the engine makes now carries the specific USCF rule that drove it — `28L1` for a single-swap rematch avoidance, `29D5` for an absolute colour-streak override, `29E1` for colour-quality tiebreaking, and so on. Citations show up in the dialog, on the existing pairing-notes column, and in the test harness diagnostics so the audit trail is consistent everywhere.
- **`docs/USCF_RULES.md` — definitive rule reference.** New in-repo document paraphrases every USCF *Official Rules of Chess* (7th edition, chapter on Swiss pairings: rules 27–29) section the FreePair engine implements. Each rule entry has a plain-language statement, the source file + member that enforces it, and the annotation reason that surfaces when the rule fires. This is the index every "?" dialog citation links back to.

## Installer

Self-contained Windows x64 installer (`FreePair-win-x64-Setup.exe`) and a no-install portable build (`FreePair-win-x64-Portable.zip`). Both bundle:

- the FreePair desktop application (Avalonia UI),
- the FreePair USCF pairing engine,
- the FIDE Dutch pairing engine (BBP Pairings),
- the .NET 10 runtime — **no separate runtime install required**.

The auto-updater (Velopack) is included. Open `Settings → Auto-update` to see your installed version and trigger an on-demand check.

## Known limitations

Unchanged from v0.6.20260604:

1. **Board-ordering preferences** — pairings, colours, and bye all match SwissSys; only the board numbering differs.
2. **SwissSys-discretionary choices that conflict with USCF rules** — SwissSys sometimes gives byes to unrated players when a rated alternative was available, or floats a non-lowest-rated player; FreePair correctly follows USCF rules. See `docs/USCF_DISCRETIONARY_DIVERGENCES.md` for the catalogue.
3. **Deferred engine improvements requiring larger refactors** — weighted conflict-cost model, absolute-board-number threading for cross-section colour tiebreaks, and floater-lock heuristic refinements. Also catalogued.
4. **"?" dialog switch diagram covers natural-group cases.** When a board's pairing involves a downfloater from a higher score group, the diagram falls back to a "group not shown" hint for this release; the text reasons still cover the decision. A future release will extend the diagram to render cross-group merges.
